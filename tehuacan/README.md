# Tehuacán combi/colectivo route data

Consolidated public transit route data for **Tehuacán, Puebla, Mexico**, collected **2026-07-14**. There is no official GTFS feed for this city; everything here was scraped from citizen-run projects and normalized. **No live vehicle telemetry exists for this system — route geometry is the ceiling of what can be known from these sources.** Schedules/frequencies below are approximations published by third parties, not operator data.

## Results at a glance

- **82 unified routes** (69 local, 13 foráneas/intercity) in `data/master_route_index.csv`
- **80 routes with geometry** (polylines, WGS84)
- **117 raw source captures**: 52 KML maps (rutastehuacan) + 65 route pages with embedded GeoJSON (queruta)
- Pseudo-GTFS: `agency.txt`, `routes.txt` (82), `shapes.txt` (80 shapes, ~25k points)
- 18 rows flagged `needs human review` — ambiguous variant matching between sources (see Notes column)

Re-run everything: `python3 scripts/01… → 07…` in order (needs `requests`; ~1 req/sec politeness delays; Moovit step was manual/browser and is not scripted).

## Web map (`map/`)

`map/index.html` + `map/routes.js` (generated by `scripts/06_build_map.py`) — a self-contained route explorer:

- **Stack:** MapLibre GL JS (open-source Mapbox fork, no API key) + OpenFreeMap vector tiles (free, no key, no usage caps) + Photon/Nominatim geocoding (OSM, free). Zero Google, zero accounts. Light/dark follows the system theme.
- **Rutas tab:** all 80 routes colored on one map; search matches route names, branded combi names («San Isidro»), and served colonias; filter local/foránea; click a route for sources + colonias.
- **¿Cómo llego? tab:** origin/destination trip search — geocodes both addresses (or pick points on the map with 📍), then matches against route geometry client-side: direct routes and one-transfer combos within ~700 m walking, ranked by walk+ride distance. No schedules or direction-of-travel data exists, so results are "which combis pass near," not timed itineraries.
- Serve it with any static server (`python3 -m http.server` in `map/`); tiles and geocoding need internet.

Name enrichment (`scripts/07_enrich_names.py`): branded names come from the route names themselves plus Moovit line descriptors (confirmed against primeralinea.com.mx's route article); plain-numbered routes are genuinely known by number. Served-colonia lists extracted from queruta.mx page text power colonia search.

## Normalization notes

- Route names differ systematically across sources ("Ruta 26 - El Riego" vs "Ruta 26"). Exact-key matching linked 24 routes; a conservative number-based merge (only when each side had exactly one candidate for that number) linked 13 more. Both names are preserved in `notes`.
- Ambiguous cases were **not** merged (e.g. rutastehuacan "Ruta 10" vs queruta "Ruta 10 Amarilla"/"Ruta 10 Verde"; the three "Ruta 32" variants) — flagged in `notes` as `needs human review`.
- Where both sources have geometry for one route, both files are listed (rutastehuacan first); GTFS shapes use the first listed file. The two geometries are independent mappings of the same route — comparing them is a built-in quality check nobody has done yet.
- The known-route checklist from the collection brief validated clean against the rutastehuacan page: all 52 accounted for (naming variants only — e.g. "Ruta 7" exists as two branded variants, "Ruta 24" is marked "(verificando)" by the source author).

## Known gaps

- **No stop locations** for most routes (Moovit's stop lists are text-only for 10 lines; OSM has 29 unassigned bus_stop nodes). `stops.txt` is deliberately absent from the pseudo-GTFS.
- **No fares or real schedules.** queruta pages mention fares on a handful of pages; Moovit's hours look templated.
- Moovit coverage was sampled, not enumerated — more Tehuacán lines are likely findable by crawling all 634 "Rutas Convencionales" lines.
- Directionality (ida/regreso) is not modeled; most source geometries are single traces or circuits.
- Sources conflict on route naming; the 18 review flags need a local person (or a ride) to resolve.
